USING_THE_CMS.SH

$ ./admin.sh --help

This is a guide to using the built-in Content Management System (CMS) for the JXVIER.COM terminal blog. The CMS provides a user-friendly interface for creating and editing content without having to manually edit markdown files.

ACCESS_CMS.SH

The CMS is accessible at:

https://jxvier.com/admin/

You can also access it by clicking the [ADMIN] link in the main navigation.

AUTHENTICATION.SH

The CMS uses Netlify Identity for authentication. Only authorized users can access the admin panel. If you're the site owner, you'll need to:

  1. Log in with your Netlify account
  2. Set up Identity service for your site
  3. Invite yourself (and any other content editors) as users

CONTENT_TYPES.SH

The CMS is configured to manage:

Blog Posts

Create and edit blog posts in the terminal style:

Fields:
- Title        // e.g., "COMMAND_LINE_BASICS.SH"
- Description  // Brief summary of the post
- Date         // Publication date
- Tags         // Categories for the post
- Body         // Main content in Markdown format

Pages

Create and edit standalone pages:

Fields:
- Title        // Page title
- Description  // META description
- Body         // Page content in Markdown
- Menu Order   // Position in navigation (if applicable)
- Permalink    // Custom URL path (optional)

Site Configuration

Edit global site settings:

Fields:
- Site Title    // "JXVIER.COM"
- Description   // "TERMINAL BLOG INTERFACE v1.0"
- Author        // Your name
- Email         // Contact email
- Social Media  // Links to profiles

WRITING_CONTENT.SH

For the terminal aesthetic:

  1. Use ALL CAPS with underscores for section headings
  2. Add .SH extension to mimic shell commands
  3. Use code blocks liberally to show "terminal output"
  4. Maintain the monospace styling with proper code formatting

Example heading:

## INSTALLATION_STEPS.SH

MEDIA_UPLOADS.SH

You can upload images through the CMS:

  1. When editing content, click the image button
  2. Select or drag an image file
  3. The CMS will upload it to /assets/uploads/
  4. The image will be optimized and available in your content

LOCAL_DEVELOPMENT.SH

When working locally, you'll need to:

# Start the 11ty development server
npx @11ty/eleventy --serve

# In another terminal, start the CMS proxy server
npx netlify-cms-proxy-server

Then access the CMS at http://localhost:8080/admin/

DEPLOYMENT.SH

Changes made in the CMS will be committed to your Git repository, triggering automatic deployments if you're using a CI/CD service like Netlify or GitHub Pages.

$ git push origin main
> Deploying changes...
> Site is now live at https://jxvier.com

Happy content editing in your terminal-inspired blog!
